# claude-progress.txt 생성 스킬

작업 진행 로그를 기록하고 관리하는 텍스트 파일을 생성합니다.

## 1. 파일 목적

- 최근 작업 내역 추적 (최대 3개)
- 작업 컨텍스트 유지
- 세션 간 연속성 제공

## 2. 파일 형식

```
# Claude Progress Log
# 최근 작업 내역을 기록합니다. (최대 3개 유지)

## 작업 기록

### [{DATE}] {TITLE}
- {DETAIL_1}
- {DETAIL_2}
- {DETAIL_3}

---

### [{DATE}] {TITLE}
- {DETAIL_1}
- {DETAIL_2}

---

### [{DATE}] {TITLE}
- {DETAIL_1}

---

(이전 작업 기록은 여기에 추가됩니다)
```

## 3. 형식 규칙

**날짜 형식:**
- `YYYY-MM-DD HH:MM` (24시간제)
- 예: `2024-12-07 15:30`

**제목:**
- 작업 내용을 간결하게 요약
- 동사로 시작 권장
- 예: `사용자 인증 기능 구현`, `버그 수정: 로그인 오류`

**세부 항목:**
- 구체적인 변경사항 나열
- 파일명이나 함수명 포함 권장
- 3-5개 항목 권장

## 4. 작업 기록 예시

```
### [2024-12-07 15:30] 사용자 인증 기능 구현
- src/auth/login.ts 생성
- JWT 토큰 발급 로직 추가
- 로그인 API 엔드포인트 구현
- 단위 테스트 작성 (auth.test.ts)

---

### [2024-12-07 10:00] 프로젝트 초기 설정
- AI 워크플로우 설정 파일 생성
- init.sh, feature-list.json 초기화
- 개발 환경 검증 완료
```

## 5. 최대 3개 유지 규칙

새 작업 추가 시:
1. 새 작업을 파일 상단에 추가
2. 작업이 3개를 초과하면 가장 오래된 작업 삭제
3. 구분선(`---`) 유지

**삭제 전:**
```
### [12-07] 작업3 (새로 추가)
### [12-06] 작업2
### [12-05] 작업1
### [12-04] 작업0 ← 삭제됨
```

## 6. 초기 생성

처음 생성 시 기본 내용:

```
# Claude Progress Log
# 최근 작업 내역을 기록합니다. (최대 3개 유지)

## 작업 기록

### [{CURRENT_DATE}] 프로젝트 설정 완료
- AI 워크플로우 설정 파일 생성
- init.sh, feature-list.json, claude-progress.txt 초기화

---

(이전 작업 기록은 여기에 추가됩니다)
```

## 7. 기존 파일 처리

`claude-progress.txt`가 이미 존재하면:
- 덮어쓰지 않음
- 기존 내용 유지
- 사용자에게 알림

## 8. 작업 추가 방법

새 작업을 추가할 때:

```bash
# 현재 날짜 가져오기
date "+%Y-%m-%d %H:%M"
```

1. 기존 파일 읽기
2. 헤더 유지
3. 새 작업을 `## 작업 기록` 바로 아래에 삽입
4. 3개 초과 시 마지막 작업 제거
5. 파일 저장

## 출력

생성 완료 시:

```
✅ claude-progress.txt 생성 완료
- 초기 작업 기록: 프로젝트 설정 완료
- 기록 가능 슬롯: 2개 남음
```

작업 추가 시:

```
✅ 작업 기록 추가됨
- 제목: {title}
- 시간: {date}
- 현재 기록: {n}/3개
```
