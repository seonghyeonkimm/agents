# claude-progress.txt 생성 스킬

작업 진행 로그를 기록하고 관리하는 텍스트 파일을 생성합니다.

## 템플릿 파일

- `templates/claude-progress.txt.template` - 전체 파일 템플릿
- `templates/progress-entry.template` - 개별 작업 기록 템플릿

## 1. 파일 목적

- 최근 작업 내역 추적 (최대 3개)
- 작업 컨텍스트 유지
- 세션 간 연속성 제공

## 2. 템플릿 사용

### claude-progress.txt.template 플레이스홀더

| 플레이스홀더 | 설명 | 예시 |
|-------------|------|------|
| `{{CURRENT_DATE}}` | 현재 날짜/시간 | `2024-12-07 15:30` |
| `{{TASK_TITLE}}` | 작업 제목 | `사용자 인증 구현` |
| `{{TASK_DETAIL_1}}` | 세부 항목 1 | `login.ts 생성` |
| `{{TASK_DETAIL_2}}` | 세부 항목 2 | `JWT 로직 추가` |
| `{{TASK_DETAIL_3}}` | 세부 항목 3 | `테스트 작성` |

### progress-entry.template

새 작업을 추가할 때 사용합니다. 동일한 플레이스홀더를 사용합니다.

## 3. 형식 규칙

**날짜 형식:**
- `YYYY-MM-DD HH:MM` (24시간제)
- 예: `2024-12-07 15:30`

**제목:**
- 작업 내용을 간결하게 요약
- 동사로 시작 권장

**세부 항목:**
- 구체적인 변경사항 나열
- 파일명이나 함수명 포함 권장
- 3-5개 항목 권장

## 4. 최대 3개 유지 규칙

새 작업 추가 시:
1. `progress-entry.template`으로 새 항목 생성
2. 파일 상단(`## 작업 기록` 바로 아래)에 삽입
3. 작업이 3개를 초과하면 가장 오래된 작업 삭제
4. 구분선(`---`) 유지

## 5. 초기 생성

`claude-progress.txt.template`을 사용하여 기본 내용 생성:
- 제목: "프로젝트 설정 완료"
- 세부: AI 워크플로우 설정 파일 생성 내역

## 6. 기존 파일 처리

`claude-progress.txt`가 이미 존재하면:
- 덮어쓰지 않음
- 기존 내용 유지
- 사용자에게 알림

## 7. 작업 추가 방법

```bash
# 현재 날짜 가져오기
date "+%Y-%m-%d %H:%M"
```

1. 기존 파일 읽기
2. `progress-entry.template` 읽기
3. 플레이스홀더 치환
4. `## 작업 기록` 바로 아래에 삽입
5. 3개 초과 시 마지막 작업 제거
6. 파일 저장

## 출력

생성 완료 시:

```
✅ claude-progress.txt 생성 완료
- 초기 작업 기록: 프로젝트 설정 완료
```

작업 추가 시:

```
✅ 작업 기록 추가됨
- 제목: {title}
- 현재 기록: {n}/3개
```
